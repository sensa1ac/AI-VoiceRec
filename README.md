# AI-VoiceRec (Smart Offline Dictation Utility)

**AI-VoiceRec** — это мощная и легковесная утилита для голосового ввода текста на русском языке, работающая на 100% локально и конфиденциально.

В отличие от классических акустических систем (например, Vosk), которые расставляют знаки препинания по длительности пауз в речи, этот проект использует современную нейросеть-трансформатор **Faster-Whisper**. Она анализирует контекст всей надиктованной фразы целиком и расставляет точки, запятые и заглавные буквы так, как это сделал бы живой человек.

---

## ✨ Ключевые особенности

* **Умная пунктуация:** Полностью автоматическая расстановка знаков препинания по смыслу, а не по таймингу пауз.
* **Абсолютный Offline:** Программа не требует интернета, не отправляет аудиоданные на сторонние сервера и работает полностью автономно.
* **0% Нагрузки на CPU в фоне:** Перехват клавиш реализован через низкоуровневые аппаратные скан-коды Windows (Win32 API). Пока хоткей не зажат, утилита «спит» и потребляет 0% процессора.
* **На лету из English в Русский:** Если вы надиктуете фразу на английском языке, модель автоматически переведет её смысл и напечатает текст на русском языке.

---

## ⚙️ Как это работает

* **Аппаратный триггер (Ctrl + Ё):** Захват клавиш привязан к физическому скан-коду кнопки (Scan code `41`). Благодаря этому горячая клавиша срабатывает безупречно и мгновенно в **любой** языковой раскладке Windows.
* **Умная запись:** Аудиопоток не пишется на жесткий диск. Пока вы удерживаете клавиши, звук напрямую собирается в оперативную память. Как только вы отпускаете клавиши, микрофон моментально отключается и освобождается в системе.
* **Инференс и безопасная вставка:** Сразу после отпускания кнопок аудио передается в нейросеть. Готовый текст безопасно инжектируется туда, где стоит курсор, через системный буфер обмена (`Ctrl+V`). Предыдущее содержимое буфера (картинки, ссылки) восстанавливается через долю секунды (0.15 сек), что дает тяжелым текстовым редакторам вроде MS Word время стабильно отобразить ввод.

### 🛡 Бронебойная обработка ошибок (Production Ready)

Архитектура и логика скрипта полностью защищены от системных лагов Windows, аппаратных сбоев и зависаний:

* **Watchdog режима сна:** При уходе компьютера в глубокий сон или на экран блокировки (Win+L) Windows часто отрывает низкоуровневые хуки приложений. Встроенный программный «сторожевой пес» отслеживает дельту времени шага цикла. Если обнаружен скачок времени, программа понимает, что ПК спал, сбрасывает мертвые дескрипторы через `kbd.unhook_all()` и мягко перезапускает слушатель клавиш без перезагрузки процесса.
* **Защита от залипания (Timeout Protection):** Если операционная система «проглотит» системное событие отпускания клавиш, запись автоматически прервется по таймауту через 3 минуты. Это страхует ОЗУ от бесконечного накопления аудиоданных.
* **Защита микрофона:** Если микрофон физически выдернут из USB-разъема или заблокирован в настройках конфиденциальности Windows, программа не падает. Поток обработки перехватывает исключение PortAudio, выводит ошибку в лог и продолжает стабильно ожидать восстановления доступа к устройству.
* **Броня буфера обмена:** Буфер обмена в Windows — разделяемый ресурс. Если в момент генерации текста буфер заблокирован другим процессом (Punto Switcher, Excel, Photoshop), утилита изолирует ошибку `Access Denied`, предотвращая краш основного потока.
* **Валидация зависимостей:** При старте программа проверяет физическое присутствие папки с весами модели. Если файлы забыли положить рядом с `.exe`, процесс выведет понятную ошибку на русском языке и безопасно завершится через 10 секунд вместо бесконечного скрытого зависания в процессах.

## 💻 Системные требования

* **ОС:** Windows 10 / Windows 11 (64-bit).
* **Процессор:** Любой современный многоядерный CPU (инференс оптимизирован под вычисления `int8`).
* **Оперативная память:** минимум 8 ГБ RAM (модель `small` занимает в памяти около 500 МБ).
* **Устройство ввода:** Любой работающий в системе микрофон.

## 🚀 Как запустить из исходного кода

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Создайте и активируйте виртуальное окружение Python:
* `python -m venv venv`
* `venv\Scripts\activate`


3. Установите необходимые зависимости:
* `pip install faster-whisper sounddevice numpy keyboard pyperclip huggingface_hub`


4. Скачайте веса модели для локальной работы, запустив скрипт инициализации (создаст папку `model_small`):
* `python download_model.py`


5. Запустите основное приложение:
* `python main.py`



---

## 📦 Компиляция в standalone-версию (EXE)

Для сборки проекта в переносимую версию используется `PyInstaller`. Команда сборки учитывает все скрытые C++ зависимости движка инференса и аудио-потока:

* `pip install pyinstaller`
* `pyinstaller --noconfirm --onedir --windowed --add-data "model_small;model_small" --collect-all faster_whisper --collect-all sounddevice main.py`

После компиляции в папке `dist/main` будет сформирована готовая сборка.

## 🛠 Настройка автозапуска (Windows Task Scheduler)

Чтобы программа автоматически запускалась при старте ПК и корректно перехватывала клавиши поверх любых окон (включая софт, запущенный от Администратора), настройте её через **Планировщик задач Windows**:

1. Создайте **Простую задачу**.
2. Вкладка **Триггеры (Triggers)** — добавьте **два триггера**:
* *При входе в систему (At log on)* — для любого пользователя. Нужен для холодного старта ПК.
* *При разблокировке рабочей станции (On workstation unlock)* — для любого пользователя. Нужен для мгновенного восстановления после сна.


3. Вкладка **Общие (General)**:
* Нажмите *Изменить пользователя* и выберите вашу текущую учетную запись (или группу Администраторы).
* Выберите радиокнопку **Выполнять только при вошедшем пользователе (Run only when user is logged on)**. *Это критично, иначе программа запустится в изолированной системной Сессии 0 без доступа к рабочему столу, клавиатуре и буферу обмена!*
* Установите флаг **Выполнять с наивысшими правами (Run with highest privileges)**.


4. Вкладка **Действия (Actions)** -> *Запуск программы*:
* **Программа/скрипт:** Укажите путь к вашему `"C:\Путь_к_папке\main.exe"` (обязательно в кавычках).
* **Рабочая папка (Start in):** Укажите путь к папке `C:\Путь_к_папке` (**строго БЕЗ кавычек и БЕЗ слэша на конце**). Если здесь будут кавычки или пустота, Python не сможет вычислить пути до весов ИИ.


5. Вкладка **Параметры (Settings)** -> Самый нижний пункт:
* **Если задача уже выполняется, применяется следующее правило:** выберите из списка **Остановить существующий экземпляр (Stop the existing instance)**.
* *Эта настройка в синергии с триггером разблокировки заставляет Windows при каждом утреннем входе в систему жестко гасить ночной «заснувший» процесс и поднимать абсолютно чистую, свежую копию утилиты.*

## 🛡 Исключения Антивируса

Поскольку утилита использует глобальные низкоуровневые хуки клавиатуры (Win32 API для скан-кодов) и программно управляет буфером обмена для эмуляции ввода, некоторые антивирусы (включая встроенный Windows Defender) могут выдать ложное эвристическое срабатывание на скомпилированный `.exe` файл.

**Настоятельно рекомендуется** добавить всю папку с распакованной программой в список исключений вашего антивируса. Это предотвратит блокировку перехвата клавиш и уберет задержки при чтении весов модели с диска.
